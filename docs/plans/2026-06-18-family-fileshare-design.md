## 一、核心需求

### 场景 1：家庭大文件传输

- 家人身处异地，跨互联网传输文件
- 痛点：聊天软件限速、限大小（1-4G）、安全检查误杀
- 需要不限大小、不限速、不扫描内容的传输

### 场景 2：远端 AI Agent 文件互通

- 家人在手机/电脑上使用 AI 助手（如 WorkBuddy），需要互传文件
- Agent 工作目录在家里的电脑上，文件需要能从其他设备发送到该目录供 Agent 处理

---

## 二、文件模型：本地云盘

与传统文件同步工具的本质区别：

| | Syncthing 模型（网上邻居） | 我们需要的模型（本地云盘） |
|------|------|------|
| 模式 | 持续全量同步 | 远程目录读写 |
| 文件存几份 | 每台设备各一份 | 只在宿主机器上一份 |
| 方向 | 自动双向 | 手动单向推/拉 |
| 心智模型 | 多台电脑共享一个文件夹 | 每台设备注册"收件箱"，其他人往里放 |

### 操作模型

```
家庭组
├── PC1 → 注册 C:\RECEIVER
├── PC2 → 注册 D:\我的资料
├── 手机 → 注册 /Photos
└── ...

手机浏览器打开 → 看到组内所有注册目录
→ 拍张照片，拖进 "PC1 的 RECEIVER" → 照片出现在 PC1 的 C:\RECEIVER
→ 点开 "PC2 的我的资料"，看到文件列表，下载需要的
```

## 三、约束条件

| 约束 | 说明 |
|------|------|
| 免费 | 完全开源，不依赖付费服务 |
| 不限大小 | 想传多大传多大 |
| 不限速 | P2P 直连，速度 = 宽带上限 |
| 不扫描 | 端到端加密，无中间服务器可查看内容 |
| 跨平台 | Windows / macOS / Linux / Android / iOS |
| 可扩展 | 家庭组人数/设备数不设硬限制 |
| 持续同步 | 保留为可选附加功能 |

---

## 四、为什么不改 Syncthing 代码

Syncthing 的核心模型是"持续全量同步共享文件夹"，与我们的"远程目录读写"模型根本冲突：

- 改动需要触及 `lib/model/`（文件夹生命周期）、`lib/config/`（配置模型）、索引交换逻辑
- 强行改造后维护困难，上游升级即可能崩溃
- 不值得为适配底层模型耗费数倍于自研的精力

## 五、为什么不全套自研

NAT 穿透、中继、TLS 加密认证是成熟领域但实现繁琐。Syncthing 免费提供了公共 STUN 服务器和中继服务器，这些基础设施可以直接使用，不需要自己搭。

## 六、最终方案：三层轻量自研

### 架构

```
┌─────────────────────────────────────────────┐
│          前端 Web UI (手机/PC 浏览器)         │
│   家庭组面板 │ 目录浏览 │ 拖拽上传 │ 下载      │
├─────────────────────────────────────────────┤
│         Hub 协调服务 (Go, 无状态)              │
│   用户/组管理 │ 目录注册表 │ 设备信令协调       │
├─────────────────────────────────────────────┤
│      Agent 设备代理 (Go, 每台设备一个)         │
│   目录暴露 │ 文件读写 │ P2P 连接 │ 中继回退    │
└─────────────────────────────────────────────┘
```

### 各层职责

**Hub（协调中心）**
- 部署在家庭常开机器或腾讯云
- 管理家庭组、用户、目录注册信息
- 信令：告知设备对方的在线状态和连接地址
- Serve 前端 Web UI
- 不传输文件数据

**Agent（设备代理）**
- 运行在每台设备上
- 启动时向 Hub 注册，上报本地暴露的目录
- 接收远程文件操作请求（列目录、上传、下载）
- 与对方 Agent 建立 P2P 直连
- 穿透失败时自动回退到公共中继

**前端**
- Web 应用，移动端自适应
- 家庭组面板：显示组内所有设备的注册目录
- 目录浏览：远程列出文件
- 拖拽/点击上传和下载

### 复用 Syncthing 基础设施（不改代码）

| 基础设施 | 用途 |
|----------|------|
| 公共 STUN 服务器 | Agent 获取自己的公网地址和 NAT 类型 |
| 公共中继服务器 | 穿透失败时的兜底传输通道 |
| 中继协议 | 参考 `lib/relay/protocol/` 实现客户端 |

### 传输速度

- 直连时速度 = min(发送方上行, 接收方下行)，无人工限速
- 中继时速度取决于中继服务器带宽，但仅作为兜底，绝大多数情况可直连
